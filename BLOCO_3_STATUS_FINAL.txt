🔄 FLUXOS IMPLEMENTADOS
───────────────────────────────────────────────────────────────────────────

FLUXO DE PAGAMENTO:
  Usuário → Clica plano → Checkout Stripe
    ↓ Preenche cartão ↓
  Stripe processa → Webhook POST
    ↓ Valida assinatura ↓
  Atualiza subscription_plan e status
    ↓
  Email confirmação enviado
    ↓
  Usuário tem acesso imediato

FLUXO DE MODERAÇÃO:
  Empresa registra (Pro/Premium) → is_verified = FALSE
    ↓
  Vagas em rascunho (não publicadas)
    ↓
  Admin vê em /admin/companies/pending
    ↓
  Admin aprova → is_verified = TRUE
    ↓
  Email enviado → Vagas agora publicadas

FLUXO DE PROTEÇÃO POR PLANO:
  Usuário tenta acessar recurso
    ↓
  Middleware valida subscription_status
    ↓
  Middleware valida subscription_plan
    ↓
  Se não tem acesso → 403 + mensagem + link upgrade
    ↓
  Se tem acesso → Próximo middleware

🚀 COMO COMEÇAR
───────────────────────────────────────────────────────────────────────────

PASSO 1: Setup Stripe (5 min)
  1. Criar conta stripe.com
  2. Gerar API keys (test mode)
  3. Criar 2 produtos (Pro, Premium)
  4. Configurar webhook

PASSO 2: Configurar .env (2 min)
  1. Adicionar STRIPE_SECRET_KEY
  2. Adicionar STRIPE_PUBLIC_KEY
  3. Adicionar STRIPE_WEBHOOK_SECRET
  4. Adicionar IDs de produtos/preços

PASSO 3: Executar Migração (2 min)
  node backend/migrations/003_add_payments_subscriptions.js

PASSO 4: Reiniciar Servidor (1 min)
  npm run dev

PASSO 5: Integrar Frontend (10 min)
  1. Importar PricingPlans.jsx em rota /pricing
  2. Importar AdminCompanyModeration em admin
  3. Adicionar links de navegação

PASSO 6: Testar (5 min)
  1. Acessar /pricing
  2. Testar checkout com cartão 4242 4242 4242 4242
  3. Admin testar moderação em /admin/companies
  4. Verificar se vagas só são publicadas após aprovação
